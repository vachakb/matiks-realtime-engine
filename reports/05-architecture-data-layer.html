05Feature surface & data model (mapped from payloads)

Categories & modes
  • BLITZ · PUZZLE · MEMORY · CLASSICAL
  • Types: DMAS, MATH_MAZE_PUZZLE, FLASH_ANZAN
  • Modes: ONLINE_SEARCH/CHALLENGE, PRACTICE, GROUP_PLAY, SURVIVAL_SATURDAY, SUMDAY_SHOWDOWN, ASYNC_CHALLENGE
  • Matchmaking falls back to bots (isBot:true)
Economy & retention systems
  • Coin staking on duels (betInfo), statikCoins, pies, XP boosters
  • Weekly leagues; unified contests + clubs (entry costs, reg forms)
  • Streaks + shields/freezers + buddy co-op streaks; quests; daily challenges + daily puzzle
  • Per-discipline ratings (Flash Anzan / Ability / Puzzle / Global), badges, referrals, chat, feed

06Two complementary tracks

Track A — the real-time engine (off-thread + prediction + clock-sync): fixes the duel itself — JS-thread saturation, match-start freeze, server-authoritative anti-cheat. Bigger build, strategic.

Track B — the data layer (this page): batch + cache + dedup + persisted queries. Fixes cold-start & bandwidth. No native code, mostly Apollo-config + server — cheaper, lower-risk, shippable fast. A founder would likely greenlight Track B first.
